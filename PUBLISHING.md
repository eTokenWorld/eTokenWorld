# Publishing Rules

**Published: 28 March 2026**

## Purpose

This repository follows a public-safe publishing model.

Its purpose is to support public understanding, ecosystem alignment, developer awareness, partnership exploration, and institutional visibility without disclosing protected technical, operational, or confidential materials.

## What May Be Published

The following categories are appropriate for this repository:

- public positioning
- project overview
- ecosystem direction
- public-safe roadmap themes
- public-safe module descriptions
- partnership and collaboration framing
- high-level developer orientation
- public FAQ and contact information

## What Must Not Be Published

The following categories must not be published in this repository:

- confidential implementation details
- proprietary execution methods
- internal scripts
- unpublished technical mechanisms
- invention-specific internal structures
- sensitive security procedures
- private deployment instructions
- credentials, tokens, keys, or signing materials
- confidential legal, financial, or business materials
- restricted partner materials

## Drafting Standard

Public materials should be:

- clear
- accurate
- high level
- public-safe
- disciplined
- partnership-ready
- understandable to developers and institutions

Public materials should avoid unnecessary technical depth where such detail could expose protected methods or confidential structures.

## Tone Standard

Publishing should aim for:

- clarity
- credibility
- seriousness
- ecosystem readiness
- partnership openness
- intellectual property discipline

Avoid:

- exaggerated claims
- hostile competitor language
- unnecessary disclosure
- confusing internal shorthand
- confidential operational detail

## Review Rule

Before publishing any file, confirm that it:

1. is public-safe
2. avoids confidential detail
3. avoids protected implementation disclosure
4. aligns with project positioning
5. supports public understanding

If there is doubt, do not publish until the material has been reviewed and rewritten.

## Repository Scope

This repository is a public-facing gateway for understanding eTokenWorld and its broader direction.

It is not a disclosure repository for protected technical implementation.
